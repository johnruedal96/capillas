# Architecture Decision Records (ADRs)

Los ADRs documentan decisiones arquitectónicas significativas del proyecto.

## ¿Cuándo crear un ADR?

- Cambios en el stack tecnológico
- Decisiones sobre patrones de arquitectura
- Elección de servicios cloud o infraestructura
- Cambios en el flujo de datos o comunicación entre servicios
- Decisiones de seguridad o compliance

## Estructura

Usamos formato [MADR 4.0](https://adr.github.io/madr/) (Markdown Any Decision Records).
Ver `0000-template.md` para la plantilla completa.

## Proceso

1. Crear archivo `docs/adr/NNNN-titulo-breve.md`
2. Usar la plantilla `0000-template.md`
3. Asignar el siguiente número secuencial
4. Abrir PR, discutir, mergear
5. Registrar en `CHANGELOG.md`
